## Use case: Авторизация на сайте(вторая попытка на второй ветке) **Цели:** Прохождение авторизации на сайте пиццерии "Большой папа" для последующей работы с учетной карточкой клиента, заказами и историей операций. **Участники:** Неавторизованный пользователь, Система **Основной сценарий:** 1. Неавторизованный пользователь заходит на сайт и нажимает кнопку "Кабинет". 2. Система открывает всплывающее окно с полями для авторизации. 3. Пользователь заполняет номер телефона и нажимает кнопку "Продолжить". 4. Система формирует код авторизации и отправляет его SMS-сообщением на номер телефона, указанный клиентом. 5. Клиент получает SMS на свой номер телефона, вводит полученный код в соответствующее поле и нажимает кнопку "Продолжить". 6. Система проверяет код на соответствие, закрывает всплывающее окно и открывает страницу учетной карточки клиента. **Альтернативные сценарии:** **Альтернативный сценарий №1: Авторизация по электронной почте** 1. Неавторизованный пользователь заходит на сайт и нажимает кнопку "Кабинет". 2. Система открывает всплывающее окно с полями для авторизации. 3. Пользователь изменяет тип поля ввода данных и заполняет ранее зарегистрированный адрес электронной почты, нажимает кнопку "Продолжить". 4. Система формирует код авторизации и отправляет его на указанный адрес электронной почты. 5. Клиент получает письмо с кодом авторизации, вводит полученный код в соответствующее поле и нажимает кнопку "Продолжить". 6. Система проверяет код на соответствие, закрывает всплывающее окно и открывает страницу учетной карточки клиента. **Альтернативный сценарий №2: Авторизация по логину и паролю** 1. Неавторизованный пользователь заходит на сайт и нажимает кнопку "Кабинет". 2. Система открывает всплывающее окно с полями для авторизации. 3. Пользователь изменяет тип поля ввода данных и заполняет ранее зарегистрированный логин, нажимает кнопку "Продолжить". 4. Система отображает поле "Пароль". 5. Клиент вводит ранее сохраненный в учетной карточке пароль и нажимает кнопку "Продолжить". 6. Система проверяет логин и пароль на соответствие, закрывает всплывающее окно и открывает страницу учетной карточки клиента.
